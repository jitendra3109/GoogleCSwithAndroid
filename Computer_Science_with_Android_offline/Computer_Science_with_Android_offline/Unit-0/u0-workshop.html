Scarne's Dice

Our second activity today will be to create a dice game called "Scarne's Dice", based on a game first described in 1945 by John Scarne. Note that the instructions for this activity are not as detailed as the previous activities. This is intended to allow you to explore AppInventor in greater details.

The game

Scarne’s Dice is a turn-based dice game where players score points by rolling a die and then:

  • score no points and lose their turn if they roll a 1, or
  • add to their points if the number rolled is 2 to 6
    • they then have to choose to either reroll or keep their score and end their turn

The winner is the first player that reaches (or exceeds) 100 points.

For example, if a player starts their turn and rolls a 6, they can choose to either ‘hold’ and end their turn, in which case they can add the 6 to their score, or to reroll and potentially score more points.

Let’s say they decide to roll again, and they get a 4. They now have the option to end their turn and add 10 points (6 + 4) to their score, or to roll again to get even more points.

They decide to roll again, but get a 1. Getting a 1 makes the player lose all the points from their turn (so their score is the same as before their turn), and finishes their turn, allowing the second player to begin their turn.

This goes on until one of the players reaches 100 points or more.

Implementing the game

Start by creating the UI shown in the image below using these images. The UI is composed of:

  • a label to display the score and status of the game
  • an image to display the current die (choose any image to be the default)
  • three buttons to roll the die; end your turn; or start over
  • a clock (which is an invisible component so won't show on the Viewer)

Then switch to the block editor and implement the following blocks:

  • Four global variables to store:

    • the user's overall score state
    • the user's turn score
    • the computer's overall score
    • the computer's turn score

  • A click handler for the "Roll" button that:

    • creates a local variable to store the current roll value (which is randomly chosen between 1 and 6).
    • update the image being shown in DiceImage to the appropriate media file (from the files provided above)
    • executes the game logic:
      • If the roll is not a 1, update the user's turn score by the value of the roll and update the label to "Your score: 0 computer score: 0 your turn score: X".
      • If the roll is a 1, reset the turn score to 0 and update the label accordingly.

At this stage you might notice that there is some functionality in your blocks that is repeated or that will be, once you start implementing the computer turn. Consider refactoring your blocks by creating procedures to:

  • Select a die roll
  • Display a die roll value as an image
  • Generate the first part of label message ("Your score: XXX computer score: XXX")

Having written the basic "Roll" functionality we can tackle the other two button handlers:

  • When ResetButton is clicked, reset the 4 global variables to 0 and update the label text
  • When HoldButton is clicked, updating the user's overall score, reset the user round score and update the label

The basic user turn functionality is now in place so you can proceed to implement the computer turn. We will start off with a very simple strategy for the computer: if the computer's round score is less than 20, re-roll, otherwise hold.

  • Create a procedure called “ComputerTurn”, it will need to:
    • Disable the roll and hold buttons
    • Create a while loop that loops over each of the computer's turn
    • During each iteration of the loop, you will need to pick a random die value (hopefully using the helper procedure you created earlier), display the die value (again, helper), and follow the game rules depending on the value of the roll.
    • Be sure to update the label with the computer's round score or "Computer holds" or "Computer rolled a one" as appropriate
  • Finally, invoke the ComputerTurn procedure from both the HoldButton handler and the RollButton handler (if the user rolled a 1)

You may find again that a helper procedure is useful in doing the house cleaning that concludes the computer's turn (updating the computer's overall score, reset its turn score and reenabling the buttons).

At this point the game should be functional so try playing a few rounds against the computer.

Although the game (hopefully) works roughly as intended you may find the computer turn to be quite hard to follow as it happens so quickly that you can hardly see the die rolls and the label updates. Let's address that by refactoring the computer turn:

  • Get rid of the while loop (but not its contents!) and make the ComputerTurn procedure handle a single roll of the computer's turn
  • In the Designer view, set the Clock1 timer to disabled by default with an interval of 500 ms
  • In the HoldButton and RollButton handlers replace the ComputerTurn invocations with a block to enable the timer
  • Create a block to handle the Clock1 Timer event by invoking the ComputerTurn procedure
  • In the ComputerTurn procedure (or its helper), when the computer's turn ends, set Clock1.TimerEnabled to false

Enjoy the wonders of a fully functional game of Scarne's dice!

Extensions

This activity (like all future activities) contains some optional extensions. If time permits, we would like each group to attempt at least one extension, either from the list below or one that you have invented yourselves.

  • Two-dice Scarne's dice:
    • Two standard dice are rolled. If neither shows a 1, their sum is added to the turn total.
    • If a single 1 is rolled, the player scores nothing and the turn ends.
    • If two 1s are rolled, the player’s entire score is lost, and the turn ends.
    • If a double is rolled, the point total is added to the turn total as with any roll, but the player is obligated to roll again
  • Implement a smarter computer player: You may find research papers about optimum solutions to this game which are quite complicated but you can don't have to go that far to make the computer player better: just take into account the difference between the computer score and user score in deciding when to hold
  • Write another dice game of your choice
  • Add support for playing the game to 100 points, declaring a winner and starting a new game
